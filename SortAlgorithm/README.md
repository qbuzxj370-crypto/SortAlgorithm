# SortAlgorithm

SortAlgorithm은 주요 정렬 알고리즘을 직접 구현하고, 내부 동작 과정을 추적(Trace)할 수 있도록 설계한 Java 기반 프로젝트입니다.

단순한 알고리즘 구현을 넘어, 구조적 설계와 테스트 가능성을 고려한 형태로 구성하는 것을 목표로 합니다.

---

## 1. 프로젝트 목적

- 주요 정렬 알고리즘 직접 구현
- 알고리즘 동작 과정 추적 기능 설계
- 공통 인터페이스 기반 확장 가능한 구조 구성
- 테스트 기반 검증 체계 마련
- 향후 성능 비교 및 분석 확장

이 프로젝트는 학습 기록과 함께, 설계 역량과 코드 구조를 명확히 드러내는 것을 목표로 합니다.

---

## 2. 패키지 구조

```
src/main/java
 ├── sort
 │    ├── Sorter.java
 │    ├── algorithms
 │    ├── trace
 │    └── util
 └── app

src/test/java
 └── sort
```

### 구성 의도

**Sorter**  
: 모든 정렬 알고리즘의 공통 인터페이스. 알고리즘 교체가 가능하도록 설계.

**algorithms**  
: BubbleSort, QuickSort 등 실제 알고리즘 구현 클래스 위치.

**trace**  
: 정렬 과정 추적을 위한 이벤트 및 리스너 구성.  
알고리즘 로직과 출력 로직을 분리하기 위한 영역.

**util**  
: 배열 생성, 검증, 보조 기능 제공.

**app**  
: 실행 및 데모를 위한 엔트리 포인트.

---

## 3. 설계 원칙

### 알고리즘과 출력 로직 분리
TraceListener 기반 이벤트 구조로 구현하며, 알고리즘 내부에서 직접 출력하지 않도록 설계.

### 확장 가능 구조
새로운 정렬 알고리즘은 Sorter 인터페이스 구현만으로 추가 가능하도록 구성.

### 테스트 중심 검증
모든 정렬 구현은 동일한 테스트 기준을 적용.  
Arrays.sort() 결과와 비교하여 정확성 검증.

### 명확성과 단순성 유지
과도한 추상화는 지양하고, 학습 목적에 적합한 구조를 유지.

## 5. Trace 설계 방향

정렬 과정은 다음과 같은 레벨로 구분하여 출력할 수 있도록 설계 예정.

- NONE : 출력 없음
- PASS : 반복 단계 단위 출력
- SWAP : 교환 발생 시 출력
- COMPARE : 비교 단계까지 출력

Trace 구현은 알고리즘 코드와 분리된 Listener 구조로 관리한다.
